# 백엔드(Backend)란 무엇인가?
## 백엔드(Backend)란 사용자가 직접 보지 못하는 웹 서비스의 뒤에서 동작하는 프로그램과 서버 부분을 말합니다.
우리가 어떤 웹사이트에 로그인하거나 글을 작성할 때 겉으로 보이는 화면은 아니지만 뒤에서 로그인 정보를 확인하고 데이터를 저장하고 필요한 정보를 가져오는 이런 작업을 처리하는 것이 바로 백엔드이다.

### 1.백엔드의 예시
사용자가 아이디와 비밀번호를 입력한다.
이 정보가 서버로 전달된다.
서버에서 데이터베이스에 저장된 정보와 비교한다.
이때 아이디와 비밀번호를 확인하는 서버 프로그램이 바로 백엔드 역할을 한다.

### 2.백엔드 개발자가 하는 일
1.서버 프로그램 개발

2.데이터베이스 관리

3.사용자 요청 처리

4.웹 서비스의 기능 구현

### 3.백엔드에서 다루는 언어
1.Java

2.Python

3.Spring Boot

4.MySQL

# 프론트엔드(Frontend) vs 백엔드(Backend)
### 1.프론트엔드
프론트엔드는 사용자가 직접 보는 화면 부분을 말합니다. 웹사이트의 디자인이나 버튼, 글자, 이미지 등 사용자가 직접 보고 조작하는 모든 요소가 포함됩니다.

#### 2.프론트엔드 에서 사용하는 주요 기술
1.HTML

2.CSS

3.JavaScript

### 3.백엔드
백엔드는 사용자가 보지 못하는 서버와 데이터 처리 부분입니다 사용자가 어떤 행동을 하면 프론트엔드에서 요청을 보내고, 백엔드는 그 요청을 처리하여 결과를 다시 보내줍니다.

1.로그인 정보 확인

2.회원 정보 저장

3.게시글 저장 및 조회

4.같은 기능을 담당한다.

#### 4.백엔드에서 주요 사용하는 기술
1.Java

2.Python

3.Spring Boot

4.MySQL

# 내가 생각하는 백엔드
##### 내가 생각하는 백엔드는 사용자가 웹사이트나 서비스를 사용할 때 보이지 않는 곳에서 중요한 기능을 처리하는 역할이라고 생각한다.
##### 예를 들어 회원가입이나 로그인, 게시글 작성과 같은 기능은 화면에서 보이지만 실제로 데이터를 저장하고 확인하는 일은 서버에서 이루어진다. 이러한 일을 담당하는 부분이 바로 백엔드이다.
##### 또한 백엔드는 단순히 데이터를 처리하는 것뿐만 아니라 웹 서비스가 정상적으로 작동하도록 관리하는 중요한 역할을 한다고 생각한다. 사용자가 많아져도 서비스가 안정적으로 운영될 수 있도록 서버와 데이터베이스를 관리하는 것도 백엔드의 역할이라고 생각한다.
##### 그래서 나는 백엔드가 사용자가 편리하게 서비스를 이용할 수 있도록 뒤에서 시스템을 관리하고 기능을 구현하는 중요한 분야라고 생각한다.
